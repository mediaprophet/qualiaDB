# Qualia Flutter — FRB API Reference

_Branch: `0.0.8-dev` | Generated by `flutter_rust_bridge` 2.12.0 | Last updated: 2026-06-07_

This document covers every function exported from `crates/qualia-flutter/rust/src/api/`
via the flutter_rust_bridge FFI layer. All functions are `async` on the Dart side (backed
by Rust's blocking thread pool) and must be `await`-ed.

Import path in Dart:

```dart
import 'package:qualia_flutter/src/rust/api/qualia_api.dart' as api;
import 'package:qualia_flutter/src/rust/api/resource_catalog.dart' as catalog;
import 'package:qualia_flutter/src/rust/api/chat_agents.dart' as agents;
```

## Current Corrections

The sections below include some older generated-era descriptions. For the current
desktop implementation on `0.0.8-dev`, treat these corrections as authoritative:

- `discoverModels()` now scans the configured local `Models/` directory for `.gguf` files.
- `getActiveModel()` / `setActiveModel()` persist the selected active model name locally.
- `launchInstalledApp()` does **not** mint a token; it resolves an entrypoint and returns either a localhost dev URL, an embedded loopback asset URL, or a `file:///` fallback.
- The shell also supports `launchInstalledAppWithContext(appName, entrypoint, surface, payloadJson, source)` for host-to-app UX handoff such as opening Anatomy from local chat.

## LLM Inference

### `runInference`

```dart
Future<String> runInference({
  required String prompt,
  required String modelPath,
})
```

Runs local LLM inference through the full Webizen-gated orchestration pipeline:

1. **Pre-flight** — `validate_intent()` checks the prompt intent against the N3Logic Rights Ontology. If denied, returns `"[Webizen blocked: <reason>]"` immediately without touching the model.
2. **Inference** — `LocalLlmAgent::infer_local_model()` runs the Phase 8 bifurcated compute loop:
   - Tokenises `prompt` using `GgufTokenizer` (parsed from the GGUF KV section; falls back to byte-level tokeniser if model absent).
   - Per decode step: embeds token → `dispatch_fused_transformer_block()` (DirectML → Accelerate → wgpu/WGSL) → sends `LogitSummary` to Webizen Sentinel over SPSC ring.
   - Sentinel injects `DenyRollback` if `anomaly_byte == 0x99`; LLM substitutes a safe token.
   - Stops at EOS or `MAX_OUTPUT_TOKENS` (2048).
3. **Post-flight** — `validate_output()` requires ≥ 1 provenance Quin. Returns `"[Webizen blocked: …]"` if ungrounded.

**Parameters**

| Name | Type | Description |
|---|---|---|
| `prompt` | `String` | The user's input text. |
| `modelPath` | `String` | Absolute path to a `.gguf` file on the device. Pass `''` to receive the "no model loaded" sentinel without invoking the engine. |

**Returns** — the generated text, or a bracketed error/block message.

**Threading** — blocks the FRB worker thread for the duration of inference (up to `INFERENCE_TIMEOUT_MS` = 30 seconds). For production use, consider wrapping in `compute()` or running from a background isolate to keep the UI thread responsive.

**Example**

```dart
final response = await api.runInference(
  prompt: 'Explain thermodynamic entropy in simple terms.',
  modelPath: '/home/user/.qualia/models/phi3-mini-q4km.gguf',
);
```

> **Note — pseudo-embeddings:** The current decode loop uses a deterministic sin-based
> embedding from the token ID rather than reading the real `token_embd.weight` tensor.
> Output text is therefore incoherent until the GGUF tensor-info section parser lands.
> The GPU compute, SPSC ring, governance pipeline, and tokeniser are all real.

## Resource Catalog

These functions read YAML catalogs from the `resources/` directory and invoke the
`qualia-cli` binary for download operations.

### `loadLlmResources`

```dart
Future<List<LLMResource>> loadLlmResources()
```

Reads `resources/llms.yaml` and returns all available LLM entries. Returns an empty list
if the file cannot be read — the UI should surface this as "catalog not found".

### `loadOntologyResources`

```dart
Future<List<OntologyResource>> loadOntologyResources()
```

Reads `resources/ontologies.yaml`.

### `downloadLlm`

```dart
Future<String> downloadLlm({required String id})
```

Invokes `qualia-cli resources download <id>` as a subprocess. The full pipeline is:
streaming fetch → `GGufSharder` pointer map → WAL provenance Quin → `CapabilityProfile`.
Returns a status message string.

### `importOntology`

```dart
Future<String> importOntology({required String id})
```

Invokes `qualia-cli resources import-ontology <id>` as a subprocess.
Pipeline: streaming fetch → `ingest::streaming_import_rdf` → WAL provenance Quin.

## Daemon & System

### `initCore`

```dart
Future<void> initCore()
```

Initialises the `qualia-client-core` app state (must be called once at startup before
any other function).

### `startDaemon`

```dart
Future<String> startDaemon()
```

Starts the Qualia semantic graph daemon on `127.0.0.1:4242`. Returns a status string.

### `daemonStatus`

```dart
Future<String> daemonStatus()
```

Queries the daemon health endpoint and returns a status string.

### `getHardwareStatus`

```dart
Future<HardwareStatus> getHardwareStatus()
```

Returns current RAM and estimated VRAM usage.

**`HardwareStatus` fields**

| Field | Type | Description |
|---|---|---|
| `ramTotalGb` | `double` | Total system RAM in GB |
| `ramUsedGb` | `double` | Used system RAM in GB |
| `vramEstimatedGb` | `double` | Estimated VRAM available |

## Qapp Vault

### `listInstalledQapps`

```dart
Future<List<String>> listInstalledQapps()
```

Returns names of qapps installed in `{storage_path}/Qapps/`.

### `launchInstalledQapp`

```dart
Future<String> launchInstalledQapp({required String qappName})
```

Reads the qapp's `qapp.json`, resolves the entrypoint, and returns a launch URL
(loopback HTTP, localhost dev server, or `file:///…`). The Flutter UI opens
localhost/127.0.0.1 URLs in `QualiaQappWebView`.

### `launchInstalledQappWithContext`

```dart
Future<String> launchInstalledQappWithContext({
  required String qappName,
  String? entrypoint,
  String? surface,
  String? payloadJson,
  String? source,
})
```

Launches a qapp with host context query params (`qualia_source`, `qualia_surface`,
`qualia_payload`, `qualia_qapp`) for chat handoff and representation surfaces.

### `generateQappCredential`

```dart
Future<String> generateQappCredential({required String qappName})
```

Generates a DID-scoped credential for the named qapp (`did:qualia:qapp:…`).

### `verifyAndInstallQapp`

```dart
Future<String> verifyAndInstallQapp({
  required String zipPath,
  required String credentialSig,
})
```

Verifies the qapp bundle's VC signature and installs it into the `Qapps/` directory.
`credentialSig` must start with `"did:qualia:qapp"`.

### `inspectInstalledQappReadiness`

```dart
Future<String> inspectInstalledQappReadiness({required String qappName})
```

Returns a JSON readiness report for required ontologies, models, and SPARQL endpoints.

## Regenerating the API

After adding or modifying any `pub fn` in `crates/qualia-flutter/rust/src/api/`:

```bash
cd crates/qualia-flutter
flutter_rust_bridge_codegen generate
# commit the resulting changes to lib/src/rust/ and rust/src/frb_generated.rs
```

The generated files (`frb_generated.dart`, `frb_generated.io.dart`,
`frb_generated.web.dart`, `frb_generated.rs`) must be committed alongside the Rust
source change. Never edit the generated files directly.
