# Qualia Flutter 0.0.10 — LLM Lifecycle, Telemetry & Backend Wiring

_Branch: `0.0.10-dev` | Release line: `0.0.10` | Last updated: 2026-06-09_

## Goal

Complete the Flutter/Rust LLM pipeline: non-blocking activation, discoverable logs, live VRAM HUD, explicit unload, and inference backend preference (Local / Remote / Hybrid).

## Phase 0 — Version & branch (this session)

| Item | Target |
|------|--------|

| Active dev branch | `0.0.10-dev` |
| Release branch | `0.0.10` (cut from dev when stable) |

## Phase 1 — Async model activation (P0)

**Problem:** `set_active_model()` mmap-probes GGUF synchronously on the FRB thread → UI freeze.

**Deliverables:**
- `set_active_model_async()` — spawns `qualia-model-activate` worker thread
- `is_model_activation_in_progress()` / `take_model_activation_error()`
- LLM Hub + Chat Environment call async API; progress via existing `LLM_LOAD|*` telemetry stream

## Phase 2 — Telemetry & logging (P0)

**Deliverables:**
- `set_telemetry_file_logging_enabled(bool)` — single source of truth (Rust flag file)
- Tray menu delegates to Rust API (no duplicate path logic)
- Settings + Dev Console toggles for file logging
- `get_telemetry_log_path()` already exposed

## Phase 3 — Live VRAM in HUD (P1)

**Deliverables:**
- Extend `EngineTelemetryFields` with `vram_used_mb`, `vram_total_mb` (DirectML DXGI on Windows)
- Extend FRB `HardwareTelemetry` + `VaultHudBar` (used / total, not just free)
- LLM Hub Overview polls shared `hardwareTelemetryProvider`

## Phase 4 — Inference backend preference (P1)

**Deliverables:**
- `inference_backend.json` in app meta (`local` | `remote` | `hybrid` + optional endpoint)
- FRB `get_inference_backend_settings` / `save_inference_backend_settings`
- Settings screen panel + LLM Hub note when not Local
- Chat agents read preference when constructing sub-agent cards

## Phase 5 — Cleanup (P2)

- Remove orphaned `hardware_telemetry_bar.dart`
- Deprecate direct `checkOllamaStatus` UI references
- Update `AGENTS.md`, `HANDOVER.md`
