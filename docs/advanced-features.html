Merkle-DAG Content Addressed History

The entire semantic graph state is versioned in a Git-like Merkle-DAG. Every commit node tracks `timestamp_ms`, `author_did`, and references a parent node, resulting in immutable historical snapshots.

  • Assertion-time snapshot queries
  • Content-addressed 32-byte SHA-256 integrity
  • O(1) multi-parent branching and merging
// Example: Navigating DagStore checkpoints

Unified V3 Volume Format

The Q42 Volume V3 unifies streaming ingest and demand-paging reads. It bakes the Merkle-DAG and Block Directories natively into the binary format, resolving format drift.

  • Fast `migrate_v2_to_v3` upgrade paths
  • Zero-overhead chunk-aligned `.bidx` directory
  • 16MB constant footprint for up to 1M blocks

Zero-Copy LoRA Mux

Dynamically swap 16+ LLM adapters with zero heap memory allocation using memmap2 weight projection and DirectML fused shaders.
