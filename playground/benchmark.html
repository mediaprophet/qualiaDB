TRANSPARENT BENCHMARK HARNESS

qualia-db vs. oxigraph / comunica / native js — live, in your browser

⚠️ Full Disclosure & Methodology
Every number on this page is produced by real JavaScript executing right now in your browser. There are no hardcoded results. The benchmark functions are fully exposed below and in the public repository.

What we can measure here: JavaScript execution speed, memory allocation via performance.memory, cold-start initialisation cost, and structural operation throughput on in-browser data structures.

What we cannot fairly measure here: Oxigraph WASM is not loaded (no public CDN bundle was available at authoring time — see caveat below). SurrealDB WASM is similarly excluded. Comunica is tested via its published npm CDN. The comparison for those engines uses equivalent data-structure operations in raw JS as a proxy, clearly labelled as such.

For full Criterion / iai bench results, see the benches/ directory and run locally with cargo bench.

DATASET & QUERY DEFINITIONS (Source of Truth)

// All tests operate on the SAME dataset, built deterministically at runtime.
DATASET_SIZE = 10,000 triples  // subject–predicate–object, random UUIDs + FNV hashes
GRAPH_EDGES = ~1.4 per node  // sparse social-graph topology
RUNS_PER_TEST = 50  // mean, p50, p95, stddev reported — not single run
QUERY_1 = Point lookup by subject ID
QUERY_2 = 2-hop neighbours (friends-of-friends)
QUERY_3 = Filter by predicate + literal value match
QUERY_4 = Ingestion: insert all 10k triples from scratch
MEM_API = performance.memory.usedJSHeapSize  // Chrome/Edge only; N/A on Firefox
TIMING = performance.now()  // sub-millisecond resolution
Qualia-DB (WASM / JS fallback) Comunica (CDN SPARQL engine) Raw JS Map (Oxigraph/SurrealDB structural proxy)
Engines NOT directly tested (and why):
• Oxigraph WASM: No stable, CORS-enabled CDN bundle was available at authoring time. To test it yourself, run cargo bench from the repo root, or follow the Oxigraph JS readme to build locally.
• SurrealDB WASM: The WASM target was experimental at time of writing; its JS heap behaviour differs significantly from the server binary.
The JS Map used as a structural proxy has similar O(1) average-case lookup to both, but does not replicate SPARQL parsing overhead or B-Tree rebalancing cost.
